検索メタデータの重み付けガイド
=================================

目的: RAG 検索（TF‑IDF / BM25 系）および埋め込み（ベクトル）検索で、どのメタデータ／本文要素がより強く検索スコアに影響するかを分かりやすくまとめる。

1. TF‑IDF / BM25（語頻度ベース） — 重みが大きい順
-------------------------------------------------
1. 本文（見出しを含む）: 単語出現頻度の母体。最も強く効く。
2. title: 要旨が凝縮されておりマッチ時に大きく寄与する。
3. 見出し / `heading_path`: セクション内の重要語をまとまって持つため上位候補に影響。
4. tags: 集中的なラベル語が高スコアを生みやすい。
5. topic / product: ドメイン語として絞り込みに効く。
6. summary / 抜粋: 本文の凝縮版として高寄与だが短文のため条件付き。
7. related_ids: 間接的な文脈拡張（直接の語頻度寄与は小さい）。
8. doc_type / status / published: 主にフィルタ用途。語重みは小さい。
9. source_path: パス名は語彙として弱く、フィルタ／再読用。
10. created_at / updated_at: 時系列フィルタや新しさ指標で用いるため語重みは最小。

実務メモ（TF‑IDF）:
- 検索精度を上げたい場合は `title` と `tags` をスコアブーストする（実装で加重）。
- 見出しで重要語が分断されないようチャンク分割を最適化する。

2. 埋め込み（セマンティック）検索 — 重みが大きい順
-------------------------------------------------
1. 本文（見出し含む）: 文脈情報が豊富で埋め込みの主材料。
2. title: 文の要旨として影響度が高い（短文だが情報密度が高い）。
3. 見出し / `heading_path`: セクション意味を明確に分離できると有効。
4. summary / 抜粋: 本文の意味を凝縮している場合は強い。
5. topic / product / tags: 補助的に意味を強化。短語は情報量が少ないため二次的。
6. related_ids: 埋め込み化すれば関連性の補助に使えるが通常は二次的。
7. doc_type / status / published / source_path / 日付類: 意味的類似性には寄与しにくく、フィルタやルール適用に用いる。

実務メモ（埋め込み）:
- 埋め込み導入時は本文を最優先に据え、`topic`/`product` はフィルタまたはスコア加算にする。
- 埋め込みは言い換え・同義表現に強いが、メタ情報はフィルタとして併用したほうが確実。

3. 運用上の推奨（短く）
----------------------
- まずやる: チャンク分割（見出し優先）を見直す。チャンクの品質が最も効く。
- 中期対応: `title` と `tags` のスコアブースト実装（TF‑IDF 側の加重）。
- ✅ 完了: ハイブリッド検索（hybrid3: キーワード＋Embedding）を導入済み。
- フィルタは `doc_type` / `published` / `product` で前処理的に適用し、誤検索を減らす。

4. 参考: 実装上の割当例（優先度に応じたブースト）
- TF‑IDF スコア = base_score * (1 + 0.8 * title_match + 0.5 * tag_match + 0.3 * topic_match)
- 埋め込み再ランク時: final_score = 0.6 * sim_score + 0.4 * (normalized_tf_score)

5. hybrid3 スコア計算式（現行）
- keyword_score = 0.8 * tfidf_score + 0.2 * bm25_score
- final_score = (1 − α) * keyword_score + α * embedding_cosine_similarity
- α = `RAG_EMBEDDING_ALPHA`（既定: 0.7）
- 埋め込みモデル: bge-m3（BAAI, 1024次元, 多言語対応）

6. アクション履歴と次の候補
- ✅ 完了: hybrid モード導入（TF-IDF 80% + BM25 20%）で既存評価セット P@1=1.000 を達成。
- ✅ 完了: hybrid3（bge-m3）統合 → 80問評価で P@1=0.912, MRR=0.956。
- 候補: `title` ブーストと見出し分割修正を行い、残り7問の失敗を改善する。
- 候補: 埋め込みモデルの fine-tuning や ANN インデックス導入で大規模化に備える。
