# 社内ネットワークに接続できない場合

## 症状
有線LANまたはWi-Fiで社内ネットワークに接続しているが、社内システムやインターネットにアクセスできない。

## 診断手順

### ステップ1：物理接続の確認
1. LANケーブルがPCとデスクのLANポートにしっかり接続されているか
2. LANポートのリンクランプ（緑色LED）が点灯しているか
3. 別のLANケーブルに交換して試す

### ステップ2：IPアドレスの確認
1. コマンドプロンプトを開く
2. `ipconfig` を実行
3. IPv4アドレスを確認：
   - `169.254.x.x` → DHCPからアドレスを取得できていない
   - `10.x.x.x` や `192.168.x.x` → 正常に取得できている

### ステップ3：IPアドレスの再取得
```
ipconfig /release
ipconfig /renew
```

### ステップ4：接続テスト
```
ping 10.0.0.1          ← デフォルトゲートウェイ
ping 8.8.8.8           ← インターネット（Google DNS）
ping www.google.com    ← DNS名前解決
```
- ゲートウェイにpingが通らない → ネットワークの物理的な問題
- 8.8.8.8にpingが通るがDNS名が解決しない → DNS設定の問題

### ステップ5：DNSキャッシュのクリア
```
ipconfig /flushdns
```

## よくある原因
- **IPアドレスの重複**: DHCPリース更新で解消。解消しない場合はIT部門に連絡
- **VLANの設定ミス**: フロア移動後に発生しやすい → IT部門にポート設定を依頼
- **ネットワーク機器の障害**: 他のメンバーも接続できない場合 → IT部門に報告
